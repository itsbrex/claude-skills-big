## Skill Building Best Practices

### Structure Requirements

**Every skill must include:**

1. **SKILL.md frontmatter**:
   ```yaml
   ---
   name: skill-name
   description: One-liner that explains what the skill does
   ---
   ```

2. **Clear instructions section**
3. **Output format templates**
4. **Best practices**
5. **Common use cases**
6. **Trigger phrases**
7. **Example requests and responses**

### Quality Standards

- **Concise but comprehensive**: 2-8 KB per skill
- **Actionable outputs**: Not just advice, actual deliverables
- **Framework-based**: Use proven methodologies
- **Examples included**: Show don't just tell
- **Business context**: Understand user's goals

### Testing Checklist

For each skill, verify:
- [ ] Skill triggers correctly from description
- [ ] Output follows specified format
- [ ] Instructions are clear and unambiguous
- [ ] Examples are realistic and helpful
- [ ] Edge cases are handled
- [ ] Output is copy-paste ready
